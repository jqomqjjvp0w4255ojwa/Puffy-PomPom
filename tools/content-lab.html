編輯三種會直接進遊戲的文字：① 性格設定（SYSTEM_PROMPT 各段）② 紙片碎片 ③ 特殊事件文案。每個分頁有自己的「上傳」按鈕，按下去就用 GitHub API 直接覆蓋對應的 data/*.json，下次部署後遊戲就會套用。

這些段落會被組進 world.js 的 SYSTEM_PROMPT。改完按最下方「上傳性格設定」。健康/飽食的行為分級在「依數值調整行為」那段。
　
提示：糰糰「健康/飽食很低卻不緊張」通常是因為這段文字沒把低數值講成危機。可在「依數值調整行為」段加重低值的描述（例如：飽食≤25 必須寫出明顯的求生行為與焦躁）。
每張紙片：關鍵字命中（scene 或玩家輸入）後，依機率掉落。match=and 代表兩邊都要命中。關鍵字用逗號分隔。
　
　
注意：id 必須唯一。已被玩家收進筆記的紙片是用 id 記錄的，盡量不要改動既有 id，新增時用新的 id。
